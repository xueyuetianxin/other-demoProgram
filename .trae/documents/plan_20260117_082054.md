## 问题分析

当前 GitHub Actions 工作流文件中的 Build Debug APK 部分代码过于繁琐，包含了过多的调试信息和复杂的错误处理逻辑。需要简化这部分代码，使其更加清晰高效。

## 解决方案

重新构建 `.github/workflows/build-android.yml` 文件中的 Build Debug APK 和 Build Release APK 部分，简化代码结构，同时保持必要的功能：

1. **简化 Build Debug APK 步骤**：
   - 移除过多的调试信息
   - 保持必要的日志记录
   - 简化错误处理
   - 保留核心功能

2. **简化 Build Release APK 步骤**：
   - 应用相同的简化原则
   - 保持与 Debug 版本的一致性

3. **保持其他必要步骤**：
   - 保留 SDK 组件安装
   - 保留环境检查
   - 保留项目结构检查

## 实施步骤

1. 编辑 `build-android.yml`，简化 Build Debug APK 部分
2. 编辑 `build-android.yml`，简化 Build Release APK 部分
3. 提交更改到 GitHub
4. 观察 GitHub Actions 构建结果

这个修改将使工作流文件更加简洁清晰，同时保持必要的功能和错误处理能力。
