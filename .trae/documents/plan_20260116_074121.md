# 修复GitHub Actions中Android SDK组件安装失败问题

## 问题分析

从GitHub Actions构建日志可以看到，"Install Android SDK components"步骤失败，错误信息为：
```
/home/runner/work/_temp/3e1523b1-1b31-4877-9f33-230f5cb4b102.sh: line 2: sdkmanager: command not found
```

**原因**：
- `sdkmanager`命令在GitHub Actions环境中找不到
- 需要先安装Android SDK Command Line Tools，然后才能使用`sdkmanager`命令
- 手动安装SDK组件的方法不可靠，容易出错

## 解决方案

使用`android-actions/setup-android`动作来自动安装所需的Android SDK组件，这个动作会自动处理SDK的安装和配置，包括sdkmanager命令。

## 执行步骤

1. 修改`build-android.yml`文件，删除手动安装SDK组件的步骤
2. 添加`android-actions/setup-android`动作，自动安装所需的Android SDK组件
3. 保留Gradle缓存步骤，提高构建速度
4. 提交并推送更改
5. 观察GitHub Actions构建日志，确认修复效果
