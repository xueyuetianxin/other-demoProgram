# 修复GitHub Actions中Check build output步骤失败问题

## 问题分析

从GitHub Actions构建日志中可以看到，"Check build output"步骤失败，错误信息为：
```
ls: cannot access 'android/app/build/outputs/apk/': No such file or directory
Error: Process completed with exit code 2.
```

**原因**：
- "Build Debug APK"步骤失败，没有生成预期的APK文件
- "Check build output"步骤尝试访问不存在的目录，导致进一步失败
- 这个额外的失败掩盖了原始的构建失败原因

## 解决方案

修改GitHub Actions工作流，调整"Check build output"步骤，添加条件判断，只有当目录存在时才执行ls命令，或者删除这个步骤，让构建失败的信息更清晰。

## 执行步骤

1. 修改`build-android.yml`文件，调整"Check build output"步骤
2. 添加条件判断，只有当目录存在时才执行ls命令
3. 提交并推送更改
4. 观察GitHub Actions构建日志，确认修复效果
