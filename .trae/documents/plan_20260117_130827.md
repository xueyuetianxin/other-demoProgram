## 问题分析

用户要求将代码恢复到打包前的状态，这意味着需要删除所有与 Capacitor 平台和 GitHub Actions 工作流相关的文件和目录，恢复原始的 React + Vite 项目结构。

## 解决方案

1. **删除 Capacitor 相关的文件和目录**
2. **删除 GitHub Actions 工作流文件**
3. **清理项目配置**
4. **恢复原始的项目结构**

## 实施步骤

1. **删除 Capacitor 平台目录**
   - 删除 `android/` 目录
   - 删除 `ios/` 目录

2. **删除 Capacitor 配置文件**
   - 删除 `capacitor.config.json`
   - 删除 `build-apk.bat` 和 `build-app.bat`

3. **删除 GitHub Actions 工作流**
   - 删除 `.github/workflows/` 目录

4. **清理 package.json**
   - 删除 Capacitor 相关的依赖项
   - 删除 Capacitor 相关的脚本

5. **恢复 vite.config.ts**
   - 确保 vite.config.ts 是原始状态

6. **提交更改**
   - 提交所有删除和修改
   - 推送更改到 GitHub 仓库

这个方案将把项目恢复到打包前的原始状态，只保留 React + Vite 项目的核心文件。
