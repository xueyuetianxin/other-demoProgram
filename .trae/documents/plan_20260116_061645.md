# 修复Android项目结构检查步骤失败问题

## 问题分析

错误原因：GitHub Actions工作流文件中"Debug Android project structure"步骤尝试访问不存在的目录`android/capacitor-android/`

- 实际项目结构中，Capacitor相关目录名称是`capacitor-cordova-android-plugins`
- 工作流文件中错误地使用了`capacitor-android`目录名称
- 这个错误的目录检查导致整个步骤失败

## 解决方案

修改GitHub Actions工作流文件，修复目录名称错误，将`android/capacitor-android/`改为`android/capacitor-cordova-android-plugins/`，或者删除这个不必要的目录检查。

## 执行步骤

1. 修改`build-android.yml`文件，修复"Debug Android project structure"步骤中的目录名称错误
2. 提交并推送更改
3. 观察GitHub Actions构建日志，确认修复效果
