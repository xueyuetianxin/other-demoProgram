# 修复GitHub Actions中Build Debug APK步骤失败问题

## 问题分析

错误原因：Java版本不兼容

- 从`capacitor.build.gradle`文件中发现，Capacitor生成的构建配置需要Java 21
  ```gradle
  android {
    compileOptions {
        sourceCompatibility JavaVersion.VERSION_21
        targetCompatibility JavaVersion.VERSION_21
    }
  }
  ```
- 但GitHub Actions工作流中设置的Java版本是17
  ```yaml
  - name: Setup Java
    uses: actions/setup-java@v4
    with:
      distribution: 'temurin'
      java-version: '17'
  ```
- 这种版本不兼容导致了Build Debug APK步骤失败

## 解决方案

修改GitHub Actions工作流文件，将Java版本从17升级到21，以匹配Capacitor的构建要求。

## 执行步骤

1. 修改`build-android.yml`文件，将Java版本从17改为21
2. 提交并推送更改
3. 观察GitHub Actions构建日志，确认修复效果
