## 问题分析

根据错误信息，**androidx.core:core:1.14.0 不存在于 Maven 仓库中**，导致构建失败：

- 搜索位置：`https://dl.google.com/dl/android/maven2/androidx/core/core/1.14.0/core-1.14.0.pom` 和 `https://repo.maven.apache.org/maven2/androidx/core/core/1.14.0/core-1.14.0.pom`
- 错误类型：`org.gradle.internal.resolve.ModuleVersionNotFoundException: Could not find androidx.core:core:1.14.0`

**依赖链分析**：
- `androidx.appcompat:appcompat:1.7.1` 依赖 `androidx.core:core-ktx:1.13.0`
- `androidx.core:core-ktx:1.13.0` 依赖 `androidx.core:core:1.13.0`

## 解决方案

将 `androidxCoreVersion` 从 1.14.0 更改为 **1.13.0**，这是：
1. 实际存在于 Maven 仓库中的版本
2. 与 compileSdk 34 兼容的版本
3. 依赖项链中实际需要的版本

## 实施步骤

1. 编辑 `android/variables.gradle` 文件
2. 将 `androidxCoreVersion` 从 1.14.0 改为 1.13.0
3. 保存文件
4. 提交更改到本地仓库
5. 推送更改到 GitHub 仓库
6. 观察 GitHub Actions 构建结果

这个方案将解决依赖项解析失败的问题，确保构建能够成功进行。
