# 修复GitHub Actions中Build Debug APK步骤失败问题

## 问题分析

- 发现`capacitor-android`模块是从`node_modules`中引用的，而不是直接存在于`android/`目录下
- `capacitor.settings.gradle`文件中定义了`capacitor-android`模块，其项目目录指向`../node_modules/@capacitor/android/capacitor`
- 工作流文件中错误地检查了`android/capacitor-android/`目录是否存在，导致步骤失败
- 之前已添加了详细的调试信息，但还需要修复目录检查问题

## 解决方案

修改GitHub Actions工作流文件，删除对不存在的`android/capacitor-android/`目录的检查，改为检查正确的`node_modules/@capacitor/android/`目录，或者删除不必要的目录检查。

## 执行步骤

1. 修改`build-android.yml`文件，修复目录检查问题
2. 简化工作流文件，删除不必要的调试步骤
3. 保留关键的构建步骤和错误处理
4. 提交并推送更改
5. 观察GitHub Actions构建日志，确认修复效果
